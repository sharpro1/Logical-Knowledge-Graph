1.The log text consists of Facts (e.g., A is a girl) and Rules (e.g., Girls have long hair).

3.The grammar of the logic formula is defined as follows:
  Predicates: P1($x, bool) ::: Does x satisfy P1?

  Rules: R1($x, True/False) >>> R2($x, True/False) ::: If x satisfies R1 (for true) or does not satisfy R1 (for false), then x satisfies R2 (for true) or does not satisfy R2 (for false).
    You can use '&&' to express AND logic, e.g.,
    R1($x, False) && R2($x, True) >>> R3($x, True)
    To express OR logic, you can split it to two rules, e.g.,
    R1($x, False) >>> R3($x, True)
    R2($x, True) >>> R3($x, True)
  Entities: Do Not use space. For example, you SHOULD use F1(cow, True) or F1(TheCow, True), but you can't use F1(The cow, True)

4.The output MUST ONLY represent the logical relationships explicitly in the log text;
  DO NOT perform logical inference to derive conclusions.
  For example, if the log contains the information 'A is a girl.' and 'Girls have long hair.', output SHOULD follow the format in point 2, like:
  Girl(A, True) ::: A is a girl.
  Girl($x, True) >>> HasLongHair($x, True) ::: Girls have long hair.
  It MUST NOT output inferred conclusions like HasLongHair(A, True).

5.When generating the Query, DO NOT reverse the logic of the question.
  For example, for the quesiton 'Is the following statement true or false? A is not a girl.',
  Query SHOULD be 'IsGirl(A, false) ::: A is not a girl', 
  Not 'IsGirl(A, True) ::: A is a girl'

6.CRITICAL RULES for valid output:

  - Each section CANNOT be empty — every section MUST have at least one entry.
  - The "Facts:" section MUST contain at least one ground fact (e.g., Furry(Anne, True)).

  - Entity names MUST NOT contain spaces. Use CamelCase: "TheCow" not "The cow", "DogP" not "Dog P".
  - The "Query:" section must contain EXACTLY ONE query in the format: Predicate(Entity, True/False)
  - It MUST NOT contain any additional text or deviations from this format.
